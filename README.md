# Rashid — Digital GIS Employee

Two-tier app demonstrating Rashid, an agentic AI for the Saudi Ministry of Municipalities and Housing (MOMAH). Combines three datasets — al-seh sub-area expansion, multi-city expansion (2024 vs 2025), and the Ash Shati Ash Sharqi 19-layer GIS analysis — into a single sidebar-driven UI.

```
Rashid Geo/
├── backend/             Express + better-sqlite3 + Anthropic SDK (port 4000)
│   ├── src/
│   │   ├── server.ts          ← entry point
│   │   ├── anthropic.ts       ← cached knowledge-base chat
│   │   ├── db/                ← schema + seed
│   │   └── routes/            ← auth, districts, knowledge, reports, chat
│   ├── data/rashid.db   ← SQLite, generated by `npm run seed`
│   └── .env             ← ANTHROPIC_API_KEY goes here
└── rashid-wireframe/    React + Vite + Leaflet + Recharts (port 5173)
    └── .env             ← VITE_API_URL=http://localhost:4000/api
```

## Quick start

```bash
# 1. Backend — install, seed, run
cd backend
npm install
npm run seed          # creates data/rashid.db with all three datasets
npm run dev           # starts on :4000

# 2. Frontend — in another terminal
cd rashid-wireframe
npm install           # already installed if you ran the wireframe before
npm run dev           # starts on :5173
```

Open [http://localhost:5173](http://localhost:5173). Log in with `demo / demo`.

## Anthropic API key

Edit [backend/.env](backend/.env):

```
ANTHROPIC_API_KEY=sk-ant-...real-key...
ANTHROPIC_MODEL=claude-sonnet-4-6   # or claude-opus-4-7, claude-haiku-4-5
```

When the key is set, free-form questions hit Claude with the full Ash Shati Ash Sharqi knowledge base as a **cached system prompt** — first turn builds the cache, subsequent turns read at ~10% input cost.

## Three data layers

| Tab | Data | Source |
|---|---|---|
| **Ash Shati Ash Sharqi** | 19 GIS layers, 9,416 features, 22 KPIs, 10 severity-coded insights, 24 keywords, 43 Q&As | [New-documents-received/](New-documents-received/) MOMAH PDFs |
| **Major Saudi Cities** | 6 cities, 2024 vs 2025 boundary expansion · synthetic 19-layer counts derived from Ash Shati ratios | Original demo + synthesized |
| **Al-Seh Sub-area** | 5-year trend (2020 → 2025): population +216%, residential +54%, density +122% | al-seh.zip / Al-Seh assessment PDFs |

## Four sections

- **Dashboard** — district dropdown (default: All Saudi), KPI grid with trend arrows, severity-coded collapsible insights section.
- **GIS** — single Saudi-wide map, district dropdown filter, right sidebar with 19 toggleable layers.
- **Ask Rashid** — split layout: chat on the left with rotating demo hint, responsive map on the right that flies to whichever district is mentioned.
- **Reporting** — generate-on-demand insights reports with area + layer dropdowns, agent-working animation, downloadable PDFs.

## Re-seed / reset

```bash
cd backend
npm run reset-db      # deletes data/rashid.db and re-runs seed
```

Every numerical fact for Ash Shati (2,676 parcels, 1,979 buildings, 778 unlicensed, 14 schools, etc.) comes directly from the MOMAH knowledge base PDFs in [New-documents-received/](New-documents-received/). For other districts the layer counts are synthesized from those Ash Shati ratios scaled by the district's area + population.
